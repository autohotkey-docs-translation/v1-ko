#Hotstring

핫스트링 옵션이나 종료 문자를 변경합니다.

#Hotstring NoMouse
#Hotstring EndChars NewChars
#Hotstring NewOptions

매개변수

NoMouse

마우스 클릭 때문에 여기에 기술하는 바와 같이 핫스트링 인식자가 재설정되지 않도록 막습니다. 부작용으로서, 이것은 또 마우스 갈고리를 핫스링이 요구하지 못하도록 막습니다 (그렇지만 스크립트가 다른 목적으로 요구하면 여전히 설치될 것입니다. 예를 들어, 마우스 핫키). #Hotstring NoMouse이 스크립트에 존재하면 모든 핫스트링에 영향을 줍니다. 단순히 물리적으로 그 아래에만 영향을 미치는 것이 아닙니다.

EndChars NewChars

단어 EndChars 다음에 스페이스를 하나 두고 다음에 새 종료 문자를 지정하십시오. 예를 들어:

#Hotstring EndChars -()[]{}':;"/\,.?!`n `t

새 종료 문자는 전체 스크립트에 전역적으로 효과가 있으므로 -- EndChars 지시어가 어디에 나타나는가에 상관 없이 -- EndChars를 한 번 이상 지정할 필요가 없습니다.

종료 문자의 최대 개수는 100개입니다. 이 길이를 넘어서는 문자는 무시합니다.

탭을 종료 문자로 만들려면, 목록에 `t를 포함하십시오. 스페이스를 종료 문자로 만들려면, 목록에 두 개의 다른 문자 사이에 포함하면 됩니다 (또는 리스트에 오직 문자가 하나 밖에 없다면 또는 하나도 없다면, 처음에 포함시킵니다).

NewOptions

핫스트링 옵션에 기술하는 바와 같이 새 옵션을 지정합니다. 예를 들어: #Hotstring r s k0 c0.

위의 EndChars와 다르게, #Hotstring 지시어는 이런 식으로 사용될 때 위치에 민감합니다. 다른 말로 하면, 핫스트링의 전체 섹션은 다음 예제와 같이 기본 옵션이 다를 수 있습니다:

::btw::by the way

#Hotstring r c  ; 이 아래의 핫스트링은 모두 "날 전송 모드"를 사용하고 기본으로 대소문자를 구분합니다.
::al::airline
::CEO::Chief Executive Officer

#Hotstring c0  ; 이 지점 아래의 모든 핫스트링을 대소문자에 무감각하게 만듭니다.

논평

다른 지시어처럼, #Hotstring는 조건적으로 실행할 수 없습니다.

관련 항목

핫스트링
